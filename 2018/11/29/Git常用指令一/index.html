Git常用指令一

##创建版本库

本地库：

>
mkdir filename –本地路径建立文件

>
cd filename
>
git init –通过此命令将这个目录变成git可以管理的代码仓库

将文件添加到本地仓库：

git add filename.txt –可反复多次使用添加多个文件至暂存区

git commit -m “输入本次提交的说明”

git status –时刻掌握仓库当前的状态

git diff –还未提交的文件做了哪些修改

版本回退等操作：

git log –查看历史修改记录

git reset –将版本回退

git reflog –将版本恢复到新版本

工作区与暂存区：

Working Directory(工作区) –在电脑内能看到的目录,即使用git init初始化的目录

Repository(版本库) –工作区内有一个隐藏目录.git，这不是工作区。而是git的版本库

工作区与版本库图示：

git add命令实际上是将所有要提交的修改放到暂存区(Stage)，git commit一次性将所有修改提交到分支

撤销修改：

git checkout – filename –丢弃工作区的修改（步骤一）

git reset HEAD filename –可以将暂存区的修改撤销掉，重新放回工作区。再执行步骤一即可撤销工作区的修改

若提交的不合适的修改到版本库时，想要撤销本次提交，参考版本回退操作。前提是没有推送到远程库。

删除文件：

git rm filename –从版本库将文件删除，并且执行一次git commit

git checkout – filename –工作区的文件删除了，可用此命令将版本库的版本替换成工作区的版本

##创建远程库
远程库：

git remote add origin git@github.com:git账户名/仓库名.git –将本地库与远程库关联

git push -u origin master –本地库的所有内容推送到远程库，首次推送加上-u参数，Git不但会把本地的master分支内容推送的远程新的master分支，还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令。

git push origin master –将master分支的最新修改推送至github

git clone –从零开发，直接创建远程库，再将远程库克隆到本地
